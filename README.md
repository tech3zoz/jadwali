# جدول عزيز (Jadwali)

تطبيق ويب شخصي (PWA) لتنظيم دوام Operation & Maintenance الدوّار، المواعيد، المهام، العادات، وأفكار التصوير — يشتغل بدون سيرفر، وكل البيانات محفوظة محلياً بالمتصفح.

A self-contained, offline-capable, installable static PWA. Vanilla HTML/CSS/JS, Arabic RTL UI, no backend.

---

## بنية المشروع / Project structure

```
.
├── index.html              # التطبيق الرئيسي (النسخة الأصلية بتبويبات سفلية)
├── css/styles.css          # كل التنسيقات
├── js/app.js               # كل المنطق (محرك الدوام، المهام، العادات، الأفكار...)
├── manifest.json           # إعدادات التثبيت كتطبيق PWA
├── service-worker.js       # التخزين المؤقت للعمل بدون إنترنت (offline)
├── icons/                  # أيقونات التطبيق (192 / 512 / maskable / apple-touch)
├── .nojekyll               # يمنع معالجة GitHub Pages للملفات
├── .github/workflows/
│   └── deploy.yml          # نشر تلقائي على GitHub Pages
└── archive/                # النسخ القديمة (ملف واحد) — محفوظة للرجوع لها
    ├── jadwali-original-singlefile.html   # النسخة الأصلية قبل التقسيم
    └── jadwali-bento.html                 # نسخة Bento Grid (بديلة)
```

> النسخة المعتمدة حالياً هي نسخة **التبويبات** (`index.html`). نسخة **Bento** محفوظة بمجلد `archive/` — لو حبيت تبدّل بينهما لاحقاً، كلّمني.

## المزايا / Features

- جدول دوام دوّار (5 أيام: صباحي / عصر / ليل / يومين إجازة) محسوب تلقائياً من تاريخ إرساء ثابت
- شبكة كالندر شهرية + عرض سريع (اليوم / ٣ أيام / أسبوع)
- مواعيد شخصية مع تصدير ملف `.ics` لإضافتها لتقويم آيفون بضغطة
- مهام مقسّمة (باقي / خلصت) مع تراجع (Undo) لمدة ٤ ثواني
- عادات بتصميم بطاقات ملونة (أيقونة + لون لكل عادة) مع **شريط أسبوع متنقّل** أعلى الشاشة (تنقّل بين الأسابيع + حلقة تقدّم لكل يوم توضح نسبة الإنجاز) — كل عادة **تتكرر بأيام محددة** من الأسبوع (مثل الجمعة فقط، أو كل سبت، أو كل يوم)، وتعرض القائمة **عادات اليوم المختار فقط** — مع أهداف قابلة للقياس اختيارية (اشرب ماي ٢٠٠٠ مل، نادي ٣ مرات) وأزرار سريعة، وستريك 🔥 يحسب الأيام المجدولة المكتملة المتتالية
- تسجيل أكل/سعرات (داخل تبويب العادات)
- أفكار تصوير لحسابين إنستقرام (تك/طبخ) مع سكربت وتاريخ نشر اختياري
- **مواقيت الصلاة** (تبويب مستقل بين العادات والأفكار): الفجر/الشروق/الظهر/العصر/المغرب/العشاء محسوبة بالكامل محلياً بدون إنترنت (طريقة أم القرى)، مع عدّاد للصلاة القادمة — الموقع (خط العرض/الطول) يُضبط من الإعدادات يدوياً أو تلقائياً عبر GPS، تقريبية (±1-2 دقيقة)
- **لغتان:** عربي / English (تبديل من ⚙️ الإعدادات، مع اتجاه RTL/LTR تلقائي)
- **مظهران:** «مبسّط» (داكن نظيف بأسلوب آبل، افتراضي) و «نجوم» (المظهر الكوني الأصلي)
- **اختيار الدوام:** كل شخص يختار مناوبته من ٥ خيارات (نفس الدورة بإزاحة مختلفة) — يظهر تلميح «اليوم: …» لتسهيل الاختيار

## التخزين / Storage

كل البيانات تُخزَّن محلياً بالمتصفح (`localStorage`) — لا يوجد سيرفر أو قاعدة بيانات خارجية.
المفاتيح: `aziz_tasks`, `aziz_habits`, `aziz_food`, `aziz_ideas`, `aziz_appointments`.
مفاتيح الإعدادات: `aziz_lang` (`en`/`ar`), `aziz_theme` (`minimal`/`starry`), `aziz_shift_phase` (`0`–`4`).

> ⚠️ البيانات مرتبطة بجهاز/متصفح واحد. تنظيف بيانات المتصفح يمسحها. (المزامنة السحابية خطوة لاحقة — انظر أسفل.)

## التشغيل محلياً / Run locally

لأن التطبيق يستخدم Service Worker، لازم يُفتح عبر خادم HTTP (مو بفتح الملف مباشرة `file://`):

```bash
# أي واحد من هذول:
python -m http.server 8000
# ثم افتح: http://localhost:8000
```

الفتح المباشر لـ `index.html` يشتغل، لكن ميزة العمل بدون إنترنت (Service Worker) ما تفعّل إلا عبر خادم أو بعد النشر.

## النشر على GitHub Pages / Deploy

المشروع جاهز للنشر التلقائي. الخطوات مرة وحدة فقط:

```bash
# 1) من داخل مجلد المشروع
git init
git add -A
git commit -m "Initial deploy: Jadwali PWA"
git branch -M main

# 2) أنشئ ريبو فاضي على GitHub باسم (مثلاً) jadwali، ثم:
git remote add origin https://github.com/<اسم-حسابك>/jadwali.git
git push -u origin main
```

3. في GitHub: **Settings → Pages → Build and deployment → Source = GitHub Actions**.
4. أي `push` بعدها ينشر تلقائياً عبر `.github/workflows/deploy.yml`.

الرابط بيكون: `https://<اسم-حسابك>.github.io/jadwali/`

> ملاحظة: كل المسارات نسبية، فالتطبيق يشتغل صح تحت مسار فرعي مثل `/jadwali/` بدون أي تعديل.

## التثبيت كتطبيق / Install as an app

- **آيفون (Safari):** افتح الرابط ← زر المشاركة ← «إضافة إلى الشاشة الرئيسية».
- **أندرويد (Chrome):** القائمة (⋮) ← «تثبيت التطبيق» / «إضافة إلى الشاشة الرئيسية».

بعد التثبيت يشتغل بملء الشاشة وبدون إنترنت.

---

## قيود معروفة / Known limitations

- إشعارات الويب الحقيقية (Push) **لا تعمل على iOS Safari** — هذا متوقع. التنبيهات الفعلية تُربط يدوياً عبر تقويم آيفون (زر تصدير `.ics`) أو Google Calendar، وهي ليست جزءاً من الكود.
- لا يوجد «منبه يرن» حقيقي ممكن تقنياً من صفحة ويب.
- البيانات على جهاز/متصفح واحد فقط (بدون مزامنة سحابية بعد).
